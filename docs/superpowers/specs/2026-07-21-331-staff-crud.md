## 1. Data Model

### Existing `staff` table (modify)

Located at `packages/database/src/schema/staff.ts`. Already exported from `packages/database/src/schema/index.ts`.

Add column:
- `profileId: uuid("profile_id").references(() => profiles.id)` ‚Äî nullable FK to auto-created login profile

Existing columns remain unchanged. Table already has soft delete (`deletedAt`).

### New `employment_contracts` table

```
id          uuid PK default gen_random_uuid()
school_id   uuid NOT NULL FK ‚Üí schools.id
staff_id    uuid NOT NULL FK ‚Üí staff.id
type        varchar(20) NOT NULL  ‚Äî 'permanent' | 'fixed_term' | 'part_time'
start_date  date NOT NULL
end_date    date  ‚Äî nullable (permanent contracts)
salary      numeric(12, 2)  ‚Äî GHS
position    varchar(100)
created_at  timestamptz default now()
```

Indexes: `(school_id)`, `(staff_id)`, composite `(school_id, staff_id)`.

Latest contract by `start_date` descending = current contract.

### Shared Types

Update `packages/shared/src/types/staff.ts` ‚Äî realign to camelCase convention (matching [3.1.3]) and match actual schema fields. Add `EmploymentContract` interface.

## 2. Service Layer

```
apps/web/services/staff/
  staff.ts                ‚Äî staff CRUD
  employment-contracts.ts ‚Äî contract CRUD
```

### `staff/staff.ts`

| Function | Description |
|----------|-------------|
| `listStaff(ctx, filters?)` | Searchable/filterable list. `filters` supports `search` (matches firstName/lastName/staffIdNumber), `department`, `role`, `status` |
| `getStaff(ctx, id)` | Single staff with latest contract |
| `createStaff(ctx, data)` | Transaction: insert staff + create `profiles` record + create initial `employment_contracts` record. Returns staff with profile. |
| `updateStaff(ctx, id, data)` | Update staff fields only (contracts managed separately) |
| `deactivateStaff(ctx, id)` | Set `status = 'inactive'` |
| `reactivateStaff(ctx, id)` | Set `status = 'active'` |

Zod schemas: `createStaffSchema`, `updateStaffSchema`.

### `staff/employment-contracts.ts`

| Function | Description |
|----------|-------------|
| `listContracts(ctx, staffId)` | All contracts for a staff member, ordered by startDate desc |
| `createContract(ctx, staffId, data)` | Insert new contract, auto-updates staff `employmentStatus` if `type` differs |

Zod schema: `createContractSchema`.

Both services throw `AppError` subclasses (`NotFoundError`, `ConflictError`).

## 3. API Routes

```
apps/web/app/api/staff/route.ts
  GET  ‚Äî listStaff with ?search, ?department, ?role, ?status
  POST ‚Äî createStaff (admin only)

apps/web/app/api/staff/[id]/route.ts
  GET    ‚Äî getStaff
  PATCH  ‚Äî updateStaff
  DELETE ‚Äî deactivateStaff (soft delete)

apps/web/app/api/staff/[id]/contracts/route.ts
  GET  ‚Äî listContracts
  POST ‚Äî createContract

apps/web/app/api/staff/[id]/reactivate/route.ts
  POST ‚Äî reactivateStaff
```

All routes wrapped with `routeHandler()` from `@/lib/api/handler`.
Auth: `requireRole('admin', 'super_admin')` on reads, `requireRole('admin')` on mutations.
